# Guia de Documentação e Estrutura do Projeto

A partir da atividade ou sistema descrito pelo usuário, siga exatamente na ordem abaixo para gerar toda a documentação e estrutura do projeto.

Para cada arquivo, crie a pasta `/.plan` e gere o arquivo solicitado com o nome indicado.

## 00 — `00_readme.md`

- Visão geral do projeto
- Objetivo do sistema
- Escopo (o que entra / não entra)
- Lista dos arquivos seguintes e ordem de leitura
- Glossário de termos
- Observações importantes para uso da documentação

## 01 — `01_funcionamento.md`

- Funcionamento da atividade/sistema solicitado
- Todas as regras de negócio
- Perfis de usuário (personas)
- Jornada do usuário completa
- Problema a resolver, oportunidades e contexto
- Cenários e exceções

## 02 — `02_requisitos_negocio.md`

- Requisitos de negócio detalhados
- Priorização (MVP / V1 / Futuro)
- Critérios de aceitação para cada requisito

## 03 — `03_requisitos_tecnicos.md`

- Ferramentas e tecnologias necessárias
- Stack definida (Django, React + Vite + TS, SQLite, etc.)
- Integrações externas, autenticação, permissões
- Restrições técnicas, desempenho, segurança
- Ambiente de desenvolvimento e variáveis de ambiente

## 04 — `04_casos_de_uso.md`

- Lista de todos os casos de uso
- Para cada caso de uso:
    - Título
    - Objetivo
    - Fluxo principal (passo a passo)
    - Fluxos alternativos
    - Fluxos de erro
    - Pré-condições e pós-condições

## 05 — `05_menu_e_navegacao.md`

- Menus e submenus do sistema
- Hierarquia completa de navegação
- Permissões por perfil de usuário
- Rotas correspondentes do frontend

## 06 — `06_modelagem_dados.md`

- Lista de todas as entidades necessárias
- Para cada entidade:
    - Lista de campos com nome, tipo, obrigatoriedade, validações
    - Chaves primárias e estrangeiras
    - Índices
    - Defaults
- Todos os relacionamentos (1:1, 1:N, N:N)
- Descrição detalhada da modelagem
- Exemplos JSON de registros

## 07 — `07_schema_prisma.md`

- `schema.prisma` completo, totalmente funcional para SQLite
- Datasource e generator corretos
- Comentários explicando cada `model`
- Instruções de uso: `prisma migrate`, `prisma generate`, `prisma db seed`

## 08 — `08_mermaid_er.md`

- Diagrama ER em Mermaid, incluindo:
    - Tabelas
    - Campos
    - Chaves
    - Relacionamentos
- Use o melhor formato Mermaid (`erDiagram` ou `classDiagram`)

## 09 — `09_api_and_rotas.md`

- Para cada entidade:
    - Listar todas as rotas RESTful
- Para cada rota:
    - Método HTTP
    - Caminho
    - Parâmetros
    - Corpo da requisição (schema)
    - Corpo da resposta (schema + exemplo)
    - Códigos de status
    - Regras de autenticação/autorização

## 10 — `10_openapi.yaml`

- Especificação OpenAPI 3 completa
- Paths, schemas, `securitySchemes`
- Consistência absoluta com o `schema.prisma`
- Exemplos completos de request e response
- Comentários para clareza

## 11 — `11_requisitos_funcionais_naofuncionais.md`

- **Requisitos funcionais:**
    - Listar cada funcionalidade esperada
    - Mapear com casos de uso e rotas
- **Requisitos não funcionais:**
    - Segurança
    - Desempenho
    - Logs e auditoria
    - Backups e recuperação
    - Escalabilidade
    - Usabilidade e acessibilidade
    - Limites de carga e tempo de resposta

## 12 — `12_plano_de_fases.md`

- Todas as fases de construção do projeto em ordem lógica:
    - Fase 0 — Preparação (repositório, CI/CD, lint, env)
    - Fase 1 — Backend básico, migrations, seeds
    - Fase 2 — API completa
    - Fase 3 — Frontend básico
    - Fase 4 — Integrações e permissões
    - Fase 5 — Ajustes finais e testes
- Para cada tarefa:
    - Nome
    - Descrição
    - Critério de aceitação
    - Status `( )` (não feito) ou `(X)` (feito)

## 13 — `13_testes.md`

- Estratégia de testes
- Ferramentas utilizadas
- Testes unitários, integração, E2E
- Planos de teste por funcionalidade
- Exemplos de testes

## 14 — `14_deploy_e_infra.md`

- Processo completo de deploy
- `Dockerfile` e `docker-compose` (se aplicável)
- Configuração de ambiente
- Variáveis de ambiente
- Estratégia de backup do SQLite
- Alternativa para migração futura (ex.: Postgres)

## 15 — `15_ci_cd.md`

- Pipelines de CI/CD (GitHub Actions ou GitLab CI)
- Stages: lint, test, build, migrate, deploy
- Regras de versionamento e releases

## 16 — `16_migrations_seeds.md`

- Estratégia de migrations
- Naming conventions
- Seeds principais
- Como executar e restaurar versões

## 17 — `17_docs_frontend_backend.md`

- **Para o backend (Django):**
    - Estrutura de apps
    - Serializers
    - Views
    - Padrões de service layer
    - Organização de URLs
- **Para o frontend (React + Vite + TS):**
    - Estrutura de pastas
    - Hooks
    - Serviços de API
    - Componentes
    - Rotas
    - Store global (se necessário)

## 18 — `18_security_privacy.md`

- Autenticação (JWT/session)
- Autorização por roles
- CSRF, CORS, XSS, SQL injection
- Logs e auditoria
- Proteção de dados pessoais
- Política de retenção

## 19 — `19_aceitacao_e_teste_de_usuario.md`

- Checklist de aceitação do sistema
- Roteiro de teste de usuário (UAT)
- Critérios de homologação

## 20 — `20_changelog_and_tasks.md`

- Modelo de CHANGELOG (semântico)
- Backlog inicial do projeto
- Modelo de issues e tarefas
- Prioridades recomendadas

## 21 — `21_modelos_extras/`

- Crie a pasta `21_modelos_extras/` contendo:
    - `postman_collection.json`
    - `env.example`
    - `.gitignore`
    - `CONTRIBUTING.md`
    - `LICENSE.md`
    - `roadmap.md`
